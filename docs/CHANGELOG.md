### Fixed
- **`/register` was CSP-dead — the self-service invite flow's page logic never ran in a CSP-enforcing browser** (`static/register.js` (new), `static/register.html`, `server/handlers.py`, `frontend/e2e/smoke.spec.js`, `tests/test_day_one_recording.py`; ensemble-evaluation finding, action-plan item 1.1). Verified two ways: a new Playwright smoke test drives `/register` in real Chromium under the production CSP — both assertions (no-invite panel swap; submit → server 403 rendered as the player-facing error line) only pass if the script executed — and a server behavior test pins the route, content type, CSP header, and no-inline-script contract. Python suite +1. **Irreversibility check:** no golden re-pin, no migration, no new `world_mutations` write path — a static file split plus one ungated GET route.

### Added
- **ADR-006 (Proposed) + ensemble action plan — the freeze challenged from the mechanics up** (`docs/decisions/ADR-006-evolving-world-with-memory.md`, `docs/roadmap/ensemble-action-plan.md`). The project owner challenged the freeze-at-launch stance directly: the goal is an *evolving* world — is the freeze truly necessary for persistence and memory? The ADR's answer, argued from verified mechanics: **no — the freeze is not fundamental; it is the integrity mechanism forced by using the generator as the world's storage medium.** The runtime regenerates the tree from the content banks on every request (`_build_world` → `generate_node_hierarchy`, no cache; no table stores nodes), and all durable history keys on `(world_seed, node_name)` — so a bank edit shifts RNG draws, renames nodes, and severs history. But the name format already separates identity from content (`<base>-<path digits>`; bank words may not contain `-`): memory needs stable identity + append-only history, **not** byte-identical regenerability. Four options laid out — A: materialize world-as-data pre-launch; **B (recommended): launch with the freeze re-worded as *scaffolding*, land an additive `nodes` mirror table verified mirror≡generation, pivot the read path post-launch, then open evolution** (banks unfreeze for new growth; deliberate change to existing nodes becomes a chronicled `WORLD_EVOLVED` event — change-as-witnessed-event serves the memory thesis better than change-made-impossible, the pattern PUZZLE_REARM already proves); C: era-gated generative evolution only; D: status quo. Status **Proposed — awaiting ratification** (interview-me practice; re-words a one-way-door covenant, so the human is the gate). The action plan sequences all eleven-persona findings into five tracks: Track 0 the evolution decision, Track 1 pre-launch gates (the `/register` CSP fix first, `FLY_API_TOKEN`, staging rehearsal), Track 2 the launch-window senses batch (hero sigil, sound invitation, pitch captures, engine-room header tucked away), Track 3 early post-launch (the pivot, evolution, cohort rhythm, since-you-left recap), Track 4 scheduled/trigger-driven — with an explicit mapping from each evaluation recommendation to its plan item (rec 3, the ceiling memo, is superseded by ADR-006: the challenge was accepted rather than documented). ### Added
- **Input moderation — a fail-open, two-tier screen on everything player-authored** (`server/moderation.py`, `consciousness.classify_content`, `server/guard.consume_moderation`, wired into `/speak`, `/agent/voice`, WS chat, and the three name surfaces — implements ADR-004 §2, the last pre-launch build). Player text is screened BEFORE it can become a chronicle row. Tier 1 is a **local filter, measured at 12–38 µs/call with zero API calls for clean input** — and only its unambiguous word-boundary blocklist can block on its own; watch words/phrases, evasion-shaped sequences (spaced/leet slurs — the trim-bypass lesson generalized), and long digit runs (doxxing shape) mark the input *ambiguous* instead, so a heuristic can never censor a player ("sniggering" escalates, it doesn't block). Tier 2 sends ambiguous inputs to **one short uncached Haiku classify** (`claude-haiku-4-5`, ~166-token system prompt — deliberately below-minimum-so-uncached, the 4096 trap dodged this time; `max_tokens=8`; hard 3 s timeout): **~$0.0003–0.0005 per escalated input by token arithmetic**, on **its own daily budget** (`consume_moderation`, default 2000/day → worst case <$1/day) so screened chat can never drain the voice budget — and `/agent/voice`'s consume was reordered (parse → screen → consume) because it used to charge the voice budget before even reading the message. **Every failure path allows** — classify error, timeout, exhausted budget, kill switch (`NESTED_WORLDS_DISABLE_MODERATION=1`), even a confused classifier reply — redaction stays the backstop; a decline is HTTP 200 in an authored world-voice line, leaves no chronicle row, no broadcast (WS senders get a private `chat_declined`, rendered by both clients so the message doesn't silently vanish), and no voice-budget charge. Registered names (`/register`, `invite mint`, `play --name`) get the STRICTER local-only screen (evasion sequences hard-block — a 32-char name has no innocent prose context); the CLI's own speak path is deliberately unscreened (the local terminal is the operator). Block/watch lists are hot-tunable via env; real production latency/cost is visible from day one via the structured `moderation_call` log line. +24 Python behavior tests (clean-costs-zero-calls, declined-speak-leaves-no-trace, chat-reaches-nobody proof via a real two-client WS session, budget/failure/kill-switch fail-open, uncached-classify-call shape) and +1 Vitest dispatch case; suites 726 → 750 and 68 → 69. No migration (the budget rides the existing `cost_budget` table), no golden re-pin, no new chronicle write path — moderation only ever *prevents* rows.
- **Self-service registration, invite-gated — the player picks their own name** (`persistence/` + migration 0012, `server/handlers.py`, `static/register.html`, `main.py` — completes ADR-004 §7's follow-up). An operator runs `invite create` to mint a **single-use registration token** (`nwr_<hex>`, `registration_tokens` table) and shares `/register?invite=<token>`; the invitee lands on a minimal in-world page, chooses the name the worlds will remember them by, and `POST /register` redeems the token. **Redemption is one transaction** (`redeem_registration_token`): token check, name-uniqueness check, play-key mint, and token consumption commit or roll back together — so a taken name replies **409 "the name 'X' is taken — choose another"** and provably leaves the token redeemable for the retry, while a successful redeem can never leave the token live (`mint_invite_key` was refactored onto a shared `_mint_invite_key_on(conn, …)` so the mint happens inside the redemption's transaction — a second writer connection there would risk `SQLITE_BUSY`). The pre-account surface is the one deliberate gate exemption with a write path: the registrant has no play key yet, so **the token is the credential**, verified in-handler; unknown, spent, and cancelled tokens all answer the same 403 (no oracle), reserved cast names and empty/oversize names are refused with the token kept, and the POST rides the standard per-IP rate limiter. Operator-named `invite mint` stays alongside; `invite tokens [--all]` audits (redeemable / "redeemed → Name" / cancelled) and `invite cancel <token>` kills a leaked unredeemed link — at most one account per leak either way (single-use). The crown behavior test drives the whole loop end-to-end: token → chosen name → minted key → that key authorizes the WebSocket and the chronicle records the CHOSEN name even when the client joins as someone else. +11 behavior tests (atomic-rollback-keeps-token, single-use, cancel semantics, no-oracle refusals, full register→play flow); suite 715 → 726. **Additive migration 0012** (new table only); no golden re-pin; the frozen `actor_identity` scheme is untouched — a self-registered player is an ordinary per-user key from the moment of minting. *(Merge-gate record — both door questions were missed at review, so the answers live here: 0012 creates one brand-new table and touches no existing table/column/row — the mildest additive case, inert if the feature were ever removed, storing no live credential. A leaked unredeemed link's blast radius is at most ONE stranger account (single-use), still named, rate-limited, and spend-capped; the move is `invite cancel nwr_<token>`, and if already redeemed, `invite tokens --all` names the account and `invite revoke` cuts it — runbook §6.)*
- **Merge gate recalibrated to fire by irreversibility; registration stays invite-gated** (`CLAUDE.md`, `docs/decisions/ADR-004-day-one-data-policy.md`). The pre-merge "quiz the human" rule was firing 5–6 comprehension questions on *every* PR, including ones the diff already answers — friction, not a gate. Recalibrated: after tests pass, write a 2–3 line **irreversibility check** yourself (does this diff re-pin a golden world, add or alter a migration, or add a `world_mutations` write path / chronicle row?) and merge on green when the answer is "none, here's why"; **escalate to an actual quiz only when that check trips a one-way door**, and then ask just the 1–2 questions it raises. The human's attention is the last gate on *irreversible* decisions — spend it there, not everywhere. Separately recorded a product decision: **self-service registration, when built, stays invite-gated** (a new player still needs an invite — the beta stays a closed, known cohort); **fully open registration is deferred** until real usage data can judge it safe for spend, moderation load, and world integrity (new ADR-004 §7 bullet + a "Revisit when…" trigger). Docs-only; no code or tests changed.
- **Unique player names, no anonymous play — names bound to the invite credential, the shared key removed entirely** (`persistence/` + migration 0011, `server/guard.py`, `server/handlers.py`, `main.py` — implements ADR-004 §7). ### Fixed
- **The `/app` scene renderer was dead under production CSP — every deployed scene was blank** (`frontend/src/main.jsx`). PixiJS v8 compiles shader/uniform glue with `new Function`, which the server's Content-Security-Policy (`script-src 'self'`, no `unsafe-eval`) rightly blocks — so `Application.init()` rejected on every production load, the silent catch ate it, and the immersive client rendered no canvas at all (it only ever worked on the CSP-less Vite dev server, which is why it kept passing local checks). Importing Pixi's `pixi.js/unsafe-eval` compatibility module replaces the eval paths; the CSP stays strict. Found because the init catch now logs a warning instead of swallowing the reason (`SceneView.jsx`).
- **Destroying a mid-init Pixi app crashed SceneView** (`frontend/src/components/SceneView.jsx`). `app.destroy()` in the effect cleanup ran while `Application.init()` was still in flight, throwing `this._cancelResize is not a function` from Pixi's resize plugin — an uncaught error with no boundary above it (React StrictMode's dev double-mount hit it every time; any fast unmount could in production). Cleanup now defers the destroy until init settles.
- **The explorer's first-load sigil was blank** (`static/explorer.js`, `static/nodeart-global.js`). The art ships as a deferred ES module, so the initial entry selection could run before `window.NodeArt` existed and nothing ever redrew. The shim now announces itself (`nodeart-ready`) and the explorer redraws the selected node's sigil the moment the art is available.
- **The WebSocket layer now speaks real RFC 6455, and one stalled client can't freeze a room** (`server/protocol.py`, `server/rooms.py`, `server/handlers.py`). Framing: unmasked client frames are rejected as protocol errors with a 1002 close (§5.1) instead of quietly accepted; fragmented messages are reassembled into one payload (§5.4) instead of each fragment being mangled into a separate "message"; pings are answered with matching pongs (§5.5.2) — including pings interleaved mid-fragmentation — and close frames are echoed so the closing handshake completes. Broadcast: each connected player now has a dedicated writer thread draining a bounded outbox, so `broadcast()` only enqueues — previously it did a blocking `sendall` per player in sequence, letting one client with a full TCP window stall every message to everyone in the room. A player whose outbox overflows is marked dead and evicted; a per-player send lock keeps reader-thread control echoes from interleaving bytes with writer-thread data frames.
- **Solved puzzles survive restarts** (`server/rooms.py::_new_session`, `persistence.get_puzzle_solve`). Co-op `PuzzleSession` state is per-process memory, but a solve is a durable fact: a fresh room now rehydrates solver + contributors from the persisted `PUZZLE_SOLVED` mutation, so a deploy no longer resets solved puzzles against a history that says they're solved. Ambient agents' solves deliberately do NOT claim the co-op session — wanderers must not lock puzzles away from players.
