## [Unreleased]

### Added
- **Non-linear entry: drop-in on first visit, resume across sessions** (`frontend/src/entry.js` (new), `frontend/src/App.jsx`, `static/explorer.js`). Traversal is non-linear — every client moves up or down from any node and there is no "reach the bottom" goal — so the clients no longer pin entry to the root. A **first-time player is dropped in at a node in the middle of the world** (one with both a parent and children, so exploration in either direction is immediate), chosen deterministically from their name via a shared FNV-1a hash, so a given player has a stable arrival point and the two clients agree on it. A **returning player resumes exactly where they left off**: the current node is persisted to `localStorage` on every move, along with the world it belonged to (seed/depth/breadth in the D3 explorer, seed in `/app`), and on load the client restores that world and reopens at the saved node — falling back to a fresh drop-in if the node is no longer present. The initial drop-in / resume position is broadcast over the WebSocket on connect (`ws.onopen` in the explorer; a `connected`-keyed effect in `/app`) so other players see the arriving player where they actually are; the D3 explorer also pans the view to centre the entry node. Pure entry-resolution logic (`findPath` / `dropInNode` / `entryPath`) lives in `frontend/src/entry.js` and is mirrored in `static/explorer.js`. Verified end-to-end in headless Chromium for both clients: a first-time player drops into a non-root node (Planet · Droven-13 for the sample world), the same player deterministically drops into the same node on a cleared browser, and a reload after moving resumes the moved-to node. New `tests/test_frontend_contract.py::TestNonLinearEntry` guards the wiring in both source clients and the built bundle.

### Changed
- **Puzzles redesigned: non-trivial, non-leaking, node-unique, difficulty-curved** (`puzzles/generators.py` (new), `puzzles/engine.py`, `tests/test_puzzles.py`). The pre-beta review found the property-keyed dynamic generators were both trivial and repetitive: answers were stated in the prompt and spelled out in the hints, ~22% of answers (139/629 in a sample tree) were a value the node already ships in its `/world` payload, and puzzles collapsed across nodes — `The Widdershins Door` (`min(3, exits)`, three possible answers ever) appeared ~150 times in one tree, and 35% of all puzzles were exact duplicates of another node. Replaced with a per-node generator: scale-themed anagrams and Caesar ciphers (answer hidden behind a transform the player reverses), inferred numeric sequences (answer hidden behind a rule), and the existing hand-written riddles/logic reused after filtering out any that leak their answer. Selection is seeded from the node's own identity, so each node's puzzle is unique to it yet reproducible — co-op players on a node see the same puzzle, and a rebuilt world regenerates it identically (`node_rng`). **Difficulty is a per-node property, not a depth curve.** Traversal is non-linear — players drop in at any node and move up or down, exploring continuously with no "reach the bottom" goal — so a depth curve would wall anyone who drops into a deep scale, make challenge yo-yo as they wander, and imply a false end-goal. Instead each node draws its own difficulty (1 gentle … 4 hard) seeded from its identity and spread across the whole range at *every* scale (`node_difficulty`); scale sets a puzzle's flavour, never how hard it is. Fairness travels with each puzzle — graduated hints, and more attempts for harder ones. The difficulty is surfaced on the `/puzzle` response and rendered as a ★ rating in both frontends so players can pick their own challenge while exploring. A selector-level guard rejects any candidate whose answer would surface in the prompt (e.g. a numeric answer colliding with the node-name index) and falls back to a guaranteed-clean anagram. Measured after the change (same sample tree, three seeds): 0 answers in prompt / hints / properties, 0 unsolvable, ~82–85% distinct puzzles with no single puzzle exceeding a small fraction of the tree, 110–124 distinct challenge types per world, and the full 1–4 difficulty spread present at both the top (Multiverse) and bottom (SubatomicParticle) scales. Server-validated attempts still never send the answer to the client. `tests/test_puzzles.py` was rewritten from per-generator formula assertions into quality invariants (no-leak, solvable-by-own-answer, graduated hints, per-node determinism, low cross-node repetition, per-node difficulty spread, and anagram/cipher transform integrity); the co-op server test now derives the expected answer through the deterministic generator instead of the old `danger_level × 2` formula.

### Fixed (pre-beta launch-readiness remediation)
- **P0 — the invited client now delivers the core loop.** The React+PixiJS app at `/app` (where invite URLs used to point) called only `/world`, `/image`, and the WebSocket — it had no way to talk to a Claude-voiced node or solve a puzzle, so the two headline mechanics were unreachable in the marketed client. Added a `frontend/src/components/Interact.jsx` panel wiring `/speak` (node conversation) and `/puzzle` + `/puzzle/attempt` (puzzle play) into the app, rebuilt the `static/app` bundle, and verified end-to-end in headless Chromium (POST `/speak` → rendered response, GET `/puzzle` → rendered prompt, POST `/puzzle/attempt` → rendered result). Invite URLs now land testers on `/` (the D3 explorer, feature-complete and WebGL-free) by default so the loop works on any device first-click (`main.py::invite_share_url`). New `tests/test_frontend_contract.py` asserts both the React source and the built bundle call the core-loop endpoints, and that the invite URL targets a playable client — the frontend↔endpoint contract that was missing.
- **P0/robustness — `/app` no longer white-screens on scene init.** `SceneView` read `app.screen` before PixiJS `Application.init()` resolved (an async race that ran on every mount), and with no error boundary an unguarded throw blanked the whole app — observed in headless Chromium and a risk on slow/mobile devices. Init is now gated behind a ready flag, `renderScene`/the ticker guard against an uninitialised or torn-down renderer, and a WebGL/WebGPU-unavailable failure leaves the scene blank while the text/speak/puzzle UI stays fully usable.
- **P1 — per-user spend sub-cap.** The daily cost cap was global-only, so one account could drain the cohort's whole budget (~500 Anthropic calls in ~25 min at 20 req/min) and degrade everyone to the quiet fallback. `server/guard.py` now enforces a per-credential daily sub-cap (`NESTED_WORLDS_ANTHROPIC_DAILY_CALLS_PER_USER` default 150, `NESTED_WORLDS_FAL_DAILY_CALLS_PER_USER` default 60) alongside the global cap; the credential is hashed into its own `cost_budget` bucket. Requests with no credential (local dev / legacy) stay global-only. Covered by `tests/test_guard.py::TestPerUserCostCap`.
- **P1 — prompt-caching claim corrected (see the "Prompt caching that actually fires" note below).** The real Opus-class minimum cacheable prefix is **4096 tokens, not 1024**. The world/agent bibles are ~1.3K tokens, so the `cache_control` marker is a silent no-op on Opus (billed as ordinary input, saves nothing) — it doesn't cost extra, but it doesn't fire either. `consciousness/__init__.py` now emits a one-time WARNING when the cached prefix is below the model minimum so the miss is visible instead of silent, and the docstring/tests carry the honest figure. Capturing the caching discount requires enlarging the bible past 4096 tokens — surfaced as a follow-up decision rather than done silently. The two tests that previously "verified" caching using the wrong 1024/3790-char threshold now guard the marker-vs-effectiveness invariant.
- **P1 — deploy contract committed.** The `Dockerfile`, `fly.toml`, and `.dockerignore` existed only as fenced code inside the deploy doc — never committed, so `fly deploy` had nothing to build and the config was untested. They're now real files with corrected sizing (1 GB RAM and connection limits above the app's WS cap for a thread-per-connection server) and a loud note that `NESTED_WORLDS_TRUST_PROXY=1` is safe only behind a proxy that overwrites `Fly-Client-IP` (a spoofed header otherwise bypasses the rate limiter). `.dockerignore` now also excludes any dev SQLite file. Guarded by `tests/test_deploy_config.py` (single always-on machine, gate-exempt `/health`, DB on the volume, WS cap ≤ proxy hard_limit, no secrets/DB in the image).
- **README/reality reconciled.** Test count corrected (316 → **349**), the two browser clients are now described accurately (both feature-complete; `/` is the default invite target), and the new per-user cost-cap env vars are documented.

### Added
- **CI workflow** (`.github/workflows/ci.yml`): GitHub Actions runs `pytest tests/ -q` against Python 3.11 with the `dev` extras installed, and a parallel job runs `npm ci && npm run build` against the `frontend/` package on every push and on PRs to `main`. Both jobs use built-in pip / npm caches keyed off the lockfiles so warm runs stay fast. This closes the "manual QA path only" gap from the beta-readiness audit — failing tests and a broken frontend build now block the merge instead of being discovered post-deploy.
- **Sentry promoted to a default dependency** (`pyproject.toml`, `server/observability.py`, `.env.example`, `README.md`): `sentry-sdk>=2.0` moved out of the `[sentry]` optional extra into the base `dependencies` list. Beta operators no longer need a second install step — setting `SENTRY_DSN` in the environment is sufficient to activate error capture. The `[sentry]` extra is removed; the warning message in `observability.setup()` now points to `pip install -e .` for the reinstall path. Behaviour with an unset DSN is unchanged (silent no-op).
- **Hosted-beta launch readiness for the 20-user cohort, with a 100-user scale plan** (`persistence/migrations/0004_invite_keys.sql`, `persistence/__init__.py`, `server/guard.py`, `consciousness/__init__.py`, `main.py`, `docs/roadmap/phase-2-scale.md`). Two pieces of operational hardening sized for the initial cohort, plus a trigger-driven scale plan that says what we do *not* build yet. (1) Anthropic concurrency semaphore: `consciousness/__init__.py` now wraps both `speak()` and `voice_agent()` in a module-level `BoundedSemaphore` whose limit comes from `NESTED_WORLDS_ANTHROPIC_CONCURRENCY` (default 8). The daily-cost cap bounds total spend; this semaphore bounds *instantaneous* concurrency so a synchronized burst from ten players speaking at once can't fire ten parallel calls and trip Anthropic's org-level RPM (which would 429 all of them and degrade /speak cohort-wide). (2) Per-user invite keys: new `invite_keys` table (migration 0004) — one row per tester with `name`, `note`, `created_at`, `revoked_at`, `last_used_at`. `server/guard.check_invite_key` now consults the shared env key AND the per-user table; either credential authorizes independently, so ops scripts can keep using the shared key while testers get individually revocable per-user URLs. `lookup_invite_key` opportunistically calls `touch_invite_key` (5-minute throttle via in-memory `_touch_cache` so the auth path doesn't fire a write per request). `invite_gate_active()` now reflects "any mechanism configured" rather than just the env var — the gate stays up as long as at least one active per-user row exists. New `python main.py invite mint|list|revoke` subcommands mint random `nw_<32-hex>` keys, list with last-used activity for "is Alice still using it?" admin signal, and revoke (idempotent — returns False on a second revoke so the CLI can give an honest message). Mint output includes the share URL `<BASE>/app?key=<KEY>&name=<NAME>` so onboarding is one click. The new `docs/roadmap/phase-2-scale.md` lays out the trigger-driven path from 20 → 100 → beyond: each deferred item (raise caps, Cloudflare in front of `/image`, Postgres switchover, Redis-backed rooms, ASGI migration) names the observable metric that justifies pulling it forward, so the operator never builds ahead of signal. Tests grew by 19 to **316 total** — 9 unit tests in `tests/test_persistence.py::TestInviteKeys` cover mint/lookup/touch/revoke/list semantics including the "revoked rows don't appear in default list" and "lookup returns None for revoked keys" invariants the auth path depends on; 7 server-level tests in `tests/test_guard.py::TestPerUserInviteKeys` exercise the gate end-to-end (per-user key via query, per-user key via header, mixed mode with shared env key + per-user keys, the gate-active reflection of DB state, and that `check_invite_key` actually advances `last_used_at`).
- **Prompt caching that actually fires** (`consciousness/__init__.py`, `tests/test_consciousness.py`): the previous structure had `cache_control` markers on a ~100-token cached prefix, but Opus 4.7 requires a ≥1024-token prefix before anything gets cached — so the markers were no-ops. Consolidated each entry point's static content into a single large cached "bible" block: `_WORLD_BIBLE` (~4400 chars, ~1200 tokens) embeds the universal preamble, world premise, all 11 level voices, behavioural rules, style guidance, and a "what to do when…" section; `_AGENT_BIBLE` (~4000 chars, ~1100 tokens) embeds the same premise plus all four persona archetypes, the 11 scales an agent passes through, and agent-specific behavior. Both are sent with `cache_control = {"type": "ephemeral", "ttl": "1h"}` — content is deploy-stable so the 1-hour TTL amortizes the cache-write cost across many calls. Per-call dynamic content (node/agent name, level, properties, history) rides as a single uncached block after the bible. New `_log_cache_usage` helper emits a structured `claude_call` line on the `nested_worlds.consciousness` logger after every Claude call, carrying `input` / `cache_read` / `cache_create` / `output` token counts so operators can verify caching is actually hitting. Tests now assert the bible embeds every level voice and every persona archetype, and that the cached prefix is ≥3790 chars (the conservative char-count needed to clear 1024 tokens at 3.7 chars/token).
- **Persistence-backend switchover plan** (`docs/decisions/ADR-003-persistence-backend.md`, `persistence/__init__.py`): documents the SQLite → Postgres migration path and concentrates the in-code seam. New `--- SQL dialect seam ---` block at the top of `persistence/__init__.py` holds the dialect-specific bits (`_NOW` constant for `datetime('now')`, `_delete_older_than(...)` helper for relative-interval `DELETE`). The three Python-side `datetime('now')` callsites (`save_agent_memory`, `upsert_ripple_score`, and `prune_mutations`'s relative-day clause) now route through these helpers; an inventory comment lists the remaining SQLite-isms (`INSERT OR REPLACE` in `save_world` and `cache_image`, plus per-backend DDL in the migration files) so they're visible without being prematurely abstracted. ADR-003 itself covers the trigger conditions for the migration, the full SQL dialect translation table, the data-copy approach, the rollback plan, and acceptance criteria for keeping the seam intact. Implementation deliberately deferred — the migration is premature today but should be a half-day refactor when the trigger fires.
- **Operational observability** (`server/observability.py`, `main.py`): three pieces aimed at running the hosted beta with eyes open. Optional Sentry integration — `SENTRY_DSN` + `pip install '.[sentry]'` enables forwarding uncaught request exceptions; unset DSN or missing SDK is a silent no-op (with a warning when DSN is set but the SDK isn't installed). `send_default_pii=False` keeps IPs and headers off the wire. Per-request structured access log emitted by `nested_worlds.access` as one JSON line carrying timestamp, hashed IP (SHA-1, 8 chars), method, path component (query string redacted so the invite key never reaches logs), status, latency in ms, and response length. `do_GET` / `do_POST` now wrap dispatch in a top-level try/except that routes uncaught exceptions through `capture_exception` and replies 500 instead of letting `BaseHTTPRequestHandler` close the connection silently. New `python main.py backup --to PATH` subcommand uses sqlite's online `Connection.backup()` to snapshot the live WAL-mode DB without pausing the server, creating parent dirs and setting 0o600 on the output for cron-style scheduling.
- **Beta hardening guardrails** (`server/guard.py`, `persistence/migrations/0003_cost_budget.sql`): six gates so a public beta URL is safe to share with invited testers. (1) Invite-key gate read from `NESTED_WORLDS_BETA_KEY`, accepted via `X-Beta-Key` header or `?key=` query param so the WebSocket connector — which can't easily set headers — still authenticates; `/health` is exempt so platform load balancers can probe without the secret. (2) Per-IP fixed-window rate limiter on `/speak`, `/agent/voice`, `/image`, `/puzzle/attempt` (default 20 req/min). Reads `X-Forwarded-For` only when `NESTED_WORLDS_TRUST_PROXY=1` so the header can't be spoofed in untrusted deploys. (3) Daily Anthropic call cap persisted in the new `cost_budget` table (migration 0003); past the cap, `/speak` and `/agent/voice` return a friendly "the worlds are quiet today" fallback instead of calling the API. (4) Daily fal.ai call cap on the same table; `/image` reports "daily image budget exhausted" past the cap. (5) Kill switches `NESTED_WORLDS_DISABLE_AI` and `NESTED_WORLDS_DISABLE_IMAGES` so the host can flip a secret without a redeploy. (6) Bounds on the four world-generator inputs (`depth`, `min_breadth`, `max_breadth`) applied centrally in `_build_world` so every endpoint that rebuilds the world tree (`/world`, `/observe`, `/puzzle`, `/puzzle/attempt`) inherits the clamp.
- **Animated presence overlays in the scene** (`frontend/src/components/SceneView.jsx`, `frontend/src/App.jsx`): when a `causal_event`, `agent_encounter`, or `puzzle_solved` arrives for the player's current node, a transient renders on top of the scene — an expanding ripple keyed off `EventKind` (gold for solves, red for failures, orange for danger, etc.), a converging-glyph encounter mark, or a four-pointed sparkle. Transients live in a separate Pixi `Container` driven by `app.ticker` so animations survive React re-renders, and are cleared on navigation. Player presence markers gained name labels and a name-hashed color palette so co-presence is legible at a glance. Closes the in-scene halves of game-design.md §"Solving the Myst Problem" rows 1 and 2 (Presence trails, Agent visibility).
- **Co-op puzzle sessions** (`server/rooms.py`, `server/handlers.py`): every `/puzzle/attempt` against the same `(seed, node_name)` now pools attempts and contributors across all players in the room. `PuzzleSession` tracks the shared attempt counter, the contributor set, and the first solver; `record_attempt()` is atomic under the room lock so concurrent solvers can't both claim. Once any player guesses correctly, the puzzle is marked solved for everyone — subsequent attempts return `result: SOLVED` with the original solver intact and the new caller appended to `contributors`. The `puzzle_solved` broadcast and the `PUZZLE_SOLVED` mutation now carry `solver` and `contributors`. The React (`frontend/src/App.jsx`) and D3 (`static/explorer.js`) clients render the solver-plus-contributors line in the event feed and pass `player_name` with their attempt. Closes the game-design.md *"Optional cooperation when players share goals"* line.
- The server now tracks puzzle attempts itself rather than trusting the client-supplied `attempt` field (which existed only as a leak guard via clamping). `correct_answer` is released on real failure (`session.attempts >= max_attempts`) and not before — a shorter, clearer security contract.
- **Test isolation** (`tests/conftest.py`): autouse fixture now also clears the global rooms registry between tests, so co-op puzzle state from one test never leaks into the next.
- **Dynamic puzzles at every level** (`puzzles/engine.py`, `puzzles/data.py`): all 11 canonical levels now resolve to property-driven generators (was 3-of-11 — Region, Universe, Planet). New generators cover Multiverse (ANAGRAM on theme), Galaxy (RIDDLE on shape), Planetary System (LOGIC on planet_count), Room (NAVIGATION on exits), Object (LOGIC on weight_kg), Molecule (LOGIC on bond_count), Atom (LOGIC on atomic_number), and SubatomicParticle (RIDDLE on particle_type). Galaxy and Particle riddles use new poetic clue maps in `puzzles/data.py` mirroring the existing `BIOME_CLUES` pattern. The static `LEVEL_POOLS` are kept as a graceful fallback for unknown levels.
- **Per-level node voicing** (`consciousness/`): `LEVEL_VOICES` catalog covers all 11 scales with distinct character notes (pronouns, time-sense, sensory vocabulary). `speak()` now sends three system blocks — universal preamble, per-level voice, per-node context — with cache breakpoints on the first two so a Region call shares its cached prefix with every other Region call. Closes the architecture-doc claim that each scale has its own register and complements the per-level visual baselines shipped in `server/imageprompt.py`.
- **Bidirectional causal propagation** (`causality/`, `multiverse/node.py`): `SpatialNode` now carries a `parent` pointer (set by `add_child` and the constructor's `children=` kwarg). `CausalityBus.propagate` gains a `direction` parameter (`down` | `up` | `both`, default `both`); origin fires exactly once, then the event cascades up the ancestor chain and/or down the subtree with the same per-hop dampening. Each fire bumps `node.ripple_score` proportional to the event's (already-dampened) strength, clamped to 1.0 — closing the README/ADR-002 callout that the field existed but was never mutated. Closes the README's *"carries effects up and down the hierarchy with dampening"* claim and lights up the seventh row of the `game-design.md` style matrix (high ripple weight → psychedelic).
- **Structured per-level image prompts** (`server/imageprompt.py`): closes ADR-002 §2 and the bulk of the game-design.md style matrix. Each of the 11 hierarchy scales gets a baseline aesthetic register from the design doc; `derive_modifiers()` adds property/history-driven mood tags (six of seven matrix rows: heavy agent activity, conflict, cooperation, pristine, corrupted, puzzle). The image cache key now folds in a short `style_signature(...)` so visuals regenerate when modifiers flip — not only when raw history count crosses a 5-event bucket.
- **Agent personas** (`agents/personas.py`): four archetypes — *tender*, *destabilizer*, *scholar*, *wanderer* — each with a voice preamble. Personas are auto-assigned deterministically from the agent's name (sha1-keyed) and can be overridden via the `persona` query/body param on `/agent`, `/observe`, and the new `/agent/voice` endpoint. The chosen persona surfaces in `AgentLog.persona`, `Agent.report()`, every causal-event payload (`{"agent": ..., "persona": ...}`), `world_mutations` rows, and the `agent_encounter` broadcast (`agent1_persona` / `agent2_persona`). Closes the README/architecture-doc claim that agents have *"distinct personalities"* and *"some destabilize; some tend."*
- **Agent voicing** (`consciousness.voice_agent`, `POST /agent/voice`): an agent speaks AS itself in its persona's voice, distinct from the existing node-voice path (`speak()`). The shared agent system preamble is prompt-cached.
- **Broader mutation recording** (`server/`, `interface/`): every interaction surface now lands in `world_mutations`, not just puzzle solves. Agent traversals (`AGENT_VISIT`, `DANGER_ALERT`) record via a per-request causality bus handler in `/agent` and `/observe`; failed puzzles record `PUZZLE_FAILED`; player chat records `PLAYER_CHAT` against the speaker's current node; `/speak` and the CLI `_speak_to` record `PLAYER_SPEAK`. Closes Phase 1 carry-over §1 — consciousness prompts and image-cache invalidation now see a richer history signal.
